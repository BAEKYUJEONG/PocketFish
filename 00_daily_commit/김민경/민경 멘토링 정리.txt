[아이디어 관련]
1. 수어 → 텍스트 변환
    - 수어를 학습시켜야하는데 웹캠 통해서 수어를 인식하는 상황들이 다양함 (조명, 카메라와의 거리, 사람마다의 손의 동작/위치) → 데이터가 상당히 많이 필요함
    - 라벨링 된 데이터가 플젝 기간동안 3~6개월동안 데이터를 수집하지 않으면 어렵다
    - 라벨링 된 데이터가 있지 않고서는 만들기 어렵다
2. 이미지 텍스트 → 수어 변환
    - 모델링 변환 작업 → 6주 안에는 어려움
    - 이미지에 있는 텍스트를 자동으로 읽기보다는 타이핑 or 음성 → 입력 받은 텍스트로 변환하는건 가능할 것 같다 (수어 영상 or 수어 연속 이미지로 변환)
3. 낚시 도감 / 다이어리 
    - 재밌는 아이템
    - 앱 출시하거나 시장에 내놓아도 재미있을 것 같은 아이템
    - 사전에 훈련된 모델이 있으면 괜찮음 , 그게 아니라면 특정 물고기의 이미지들을 크롤링해서 한 종 / 두 종 정도 학습시켜서 해보면 상당히 재밌는 서비스가 될 수 있을 것 같음
4. 미션 인증
    - 특정 장소 파악? → 그 지역의 랜드마크 명칭이 나오진 않을것같음, 랜드마크 명칭 파악 과정의 시간이 많이 소요되지 않을까
5. 이미지 기반 번역기
    - 구현이 가장 쉬울것같음
    - 많이 나와있는 서비스
    - 구현에 재미요소는 없을 것 같음
    - 경험 쌓는데는 도움이 되겠지만, 신선도는 떨어진다. '이런 고민을 해보았다' 이런 부분으로는 부족할 것 같다.

    Q2.
    역할분담
    현장에서는 연구자 / 모델 개발 / 응용 개발
    서비스 개발에서 프론트 / 백으로 나뉨
    ai는 다같이 참여하는게 좋음 → 나중의 면접을 위해서도 다른 분들의 경험을 나눌 수 있어야 함
    백엔드 개발자를 구하지만 프론트도 할 줄 아는 개발자를 원함

    ai를 활용한 응용 개발자로 취업하는 것을 추천

    Q4.
    네이티브로 개발하는 니즈가 가장 큼 (그 다음이 하이브리드 앱, 그 다음이 리액트/플러터)
    현재는 그런데 멀티 플랫폼으로 퍼블리싱할 수있는 리액트 네이티브나 플러터 같은 앱들이 중국에서 각광 받을 것임 , 니즈도 점점 늘어나고있음
    리액트 네이티브 실제로 쓰는 부서도 있음, 프로덕트로 나가는 부분들로는 많지는 않음
    사내 직원 대상 앱이거나 내부용 앱들은 리액트 네이티브로 활용하고 있음
    기능적, 성능적 제약 → 속도면이나 퍼포먼스 측면에서는 차이가 많이 남
    퍼포먼스가 중요한 앱에는 치명적임 (그런 앱들은 네이티브 쓰는게 좋음 그렇지 않다면 내부 인프라 or 서비스에서는 많이 사용됨)
